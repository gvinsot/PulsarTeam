Containers are started lazily per agent when a tool call needs execution: `AgentManager` invokes `sandboxManager.ensureSandbox(...)`, which runs `docker run -d` with the sandbox image, resource limits, mounts, and git env vars, then configures git and clones the project into `/workspace/<project>`. Tool actions run via `docker exec` inside that container. Existing healthy containers are reused; dead or mismatched ones are recreated; all are cleaned on shutdown and orphan cleanup at startup.
